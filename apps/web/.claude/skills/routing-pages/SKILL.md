# Routing & Pages

TanStack Router configured for **file-based routing rooted at `src/pages/`** (not the default
`src/routes/`) — the router plugin is configured accordingly in `vite.config.ts`. Every route
file exports a `Route` created with `createFileRoute(path)`.

## Structure

```
pages/
├── __root.tsx                              # root route, wraps everything
├── session.tsx                             # /session (public — login)
├── signup.tsx                              # /signup (public)
└── _authenticated/                         # pathless layout route — auth guard
    ├── layout.tsx                          # shell: header + scroll area + Outlet
    ├── account/
    │   ├── layout.tsx
    │   ├── index.tsx
    │   ├── profile.tsx
    │   └── preferences.tsx
    ├── (dashboard)/index.tsx                # route group — doesn't affect the URL
    └── $workspaceId/                        # dynamic segment
        ├── index.tsx
        ├── $.tsx                            # splat route
        └── settings/
            ├── layout.tsx
            ├── index.tsx
            ├── general.tsx
            ├── members.tsx
            ├── roles.tsx
            └── activity.tsx
```

- `_authenticated` (leading underscore) is a **pathless layout route** — it doesn't add a URL
  segment, it just wraps its children and can gate access.
- `(dashboard)` (parentheses) is a **route group** — organizational only, doesn't affect the URL
  either.
- `$workspaceId` is a dynamic path segment; `$.tsx` is a splat/catch-all route.
- `src/route-tree.gen.ts` is auto-generated by the router plugin from this file structure —
  **never edit it by hand** (it's already excluded from lint).

## Root route

```typescript
// pages/__root.tsx
export const Route = createRootRouteWithContext<{
  auth: ReturnType<typeof useAuth>
}>()({
  component: RootComponent,
})
```

The router context carries `auth` (see the `data-fetching` skill for `useAuth`), so any route's
`beforeLoad` can read `context.auth`.

## Layout routes

A layout route renders shared chrome and an `<Outlet />` for its children:

```typescript
export const Route = createFileRoute('/_authenticated')({
  component: RouteComponent,
})

function RouteComponent() {
  return (
    <div>
      <Header />
      <Outlet />
    </div>
  )
}
```

## Auth guard pattern

Gate an entire subtree by redirecting in `beforeLoad` on the layout route — not per-page:

```typescript
export const Route = createFileRoute('/_authenticated')({
  beforeLoad: ({ context, location }) => {
    const { isAuthenticated } = context.auth
    if (!isAuthenticated)
      throw redirect({
        to: '/session',
        search: { redirect: location.href },
      })
  },
  component: RouteComponent,
})
```

## Leaf pages

A leaf page is typically thin — it reads route params via `Route.useParams()` and composes
feature components from `components/app/`:

```typescript
export const Route = createFileRoute(
  '/_authenticated/$workspaceId/settings/general',
)({
  component: RouteComponent,
})

function RouteComponent() {
  const { workspaceId } = Route.useParams()
  return (
    <div className='flex flex-col gap-8'>
      <GeneralSection workspaceId={workspaceId} />
      <Separator />
      <DangerSection workspaceId={workspaceId} />
    </div>
  )
}
```

Business logic and data fetching live in the composed section components (see the
`data-fetching` skill), not in the route file itself.
