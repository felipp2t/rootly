# Rootly Web

React frontend for Rootly, built with Vite, TanStack Router/Query/Form and shadcn/ui.

## Stack

| Concern | Technology |
|---|---|
| Framework | React 19 + Vite |
| Routing | TanStack Router (file-based, `src/pages/`) |
| Data fetching | TanStack Query, over a client generated from the API's OpenAPI spec |
| Forms | TanStack Form + Zod |
| UI | shadcn/ui + Tailwind CSS 4 + Radix primitives |
| Theming | `next-themes` (light/dark) |
| HTTP client | Axios, generated via [orval](https://orval.dev) |

## Getting started

Requires the [API](../api) running locally at `http://localhost:3333` (see its README).

```bash
# from the repo root
pnpm install
pnpm dev:web
```

The app runs at `http://localhost:5173`.

## Commands

```bash
pnpm dev            # start the dev server
pnpm build           # type-check and build to dist/
pnpm preview         # preview the production build locally
pnpm lint            # Biome check
pnpm format          # Biome format --write
pnpm generate        # regenerate the API client from the API's OpenAPI spec
```

## API client

`src/api/` is entirely generated by `orval` from the API's `openapi.json` — never edit it by
hand. After changing an endpoint on the API side, regenerate it with:

```bash
pnpm generate
```

This also regenerates `openapi.json` on the API first, so the two stay in sync. Generated files
(`src/api/**`, `src/route-tree.gen.ts`) are excluded from linting for the same reason.

## Structure

```
src/
├── api/           # generated HTTP client + Zod-inferred types (orval)
├── pages/         # file-based routes (TanStack Router)
├── components/
│   ├── ui/         # shadcn/ui primitives
│   └── app/        # feature components (settings, activity feed, etc.)
├── hooks/
├── lib/
└── utils/
```

## Known limitation

The API base URL defaults to `http://localhost:3333` in a few places (`src/lib/env.ts`,
`src/lib/fetch.ts`) rather than being fully driven by `VITE_API_URL` everywhere — fine for local
development, but worth centralizing before deploying this to a non-local environment.
