## Routing
TanStack Router, file-based under `src/pages` (e.g. `_authenticated/$workspaceId/settings/*`). Route tree is auto-generated into `src/route-tree.gen.ts` — never hand-edit; it regenerates from the Vite plugin on dev/build.

## API client — fully generated, never hand-edit
`src/api/**` (grouped by tag: `auth`, `folders`, `items`, `members`, `roles`, `workspaces`, `notifications`, `invites`, `me`, plus `model/` for schemas) is generated by **orval** (`apps/web/orval.config.ts`) from the API's OpenAPI spec, in `tags-split` + `react-query` mode, using a custom axios mutator at `src/lib/fetch.ts` (`fetchWithAuth`).
Regenerate with `pnpm --filter web generate` (this also re-runs the API's `openapi` script first, so the API must build/typecheck cleanly). Do this after any API route or Zod schema change.

## Components
- `src/components/ui/*` — shadcn/ui primitives (button, dialog, dropdown-menu, etc.).
- `src/components/app/*` and top-level files (`header.tsx`, `item-card.tsx`, `folder-card.tsx`, `notification*.tsx`, `workspace-card.tsx`) — feature/domain components composed from the primitives.
